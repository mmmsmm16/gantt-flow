# feature/release-ready — 評価ガイド

このブランチは「自分（Claude）だったらここまで仕上げてリリースする」という観点で、`main` の上に
**体験・機能・アクセシビリティ・堅牢性**を積み増したものです。設計の核（単一データ → 工程表とフロー図を
`reconcileFlow` で同期 / `packages/core` は UI 非依存）は壊さず、その上に載せています。
**まるごと採用しても、部分的に拾っても**構いません。

## 触ってみる

```bash
npm install            # 未インストールなら
npm run dev -w @gantt-flow/desktop      # http://localhost:5173
```

起動したら **「サンプルを開いて試す」** を押すと、受注〜出荷業務のサンプル（4部門レーン・階層・前後関係・
I/O・課題）が入った状態で始められます。

### まず試してほしい操作
- **Ctrl/⌘+K** … コマンドパレット。工程名/工程No で検索ジャンプ、各種操作を実行。
- **?** … キーボードショートカット一覧。
- フロー右上の **整列** … 依存で段組み＋担当レーンで自動配置。並行する工程があるレーンは自動で太くなる。
- レーン名の下辺を **ドラッグ** … レーンの高さを手動で変更（下のレーンも連動）。
- 矢印/制御ノード/付箋を **クリック→Delete** … 削除。矢印は**ダブルクリック**で分岐ラベル。
- 何か編集して **タブを閉じる→再度開く** … 「前回の作業を復元しますか？」（自動退避からの復旧）。

## 設計上の約束を守った点

- `packages/core` は UI/OS 非依存のまま（Node 単体テスト可）。
- コア変更は commands 経由の純粋関数、flow への反映は `reconcileFlow` 経由のみ。ID は注入 `idGen`。
- reconcile の不変条件（対象1⇄ノード1 / データ編集で x/y 不変 / pinned 保持 / 冪等 / ダングリング無し）を維持。
  可変レーン高さは **整列時のみ**太くし、reconcile では座標・高さを動かさない（不変条件を守るため）。
- 保存形式は後方互換（`Swimlane.height?` は任意追加でマイグレーション不要。旧 `.json` はそのまま開ける）。
