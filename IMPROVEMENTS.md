# 積み残し改善 TODO（優先度付き）

今回の総合改善（品質監査 50 件＋UI/UX 改善カタログ 4 件＋新機能 3 本＝業務リント/検証パネル・ヒアリング状況・改善効果レポート）で
**着手せず残した項目**を、次回以降のために優先度付きで記録する。凡例: 優先度＝高（次に着手すべき）/ 中（余力があれば）/ 低（現状維持で可・掃除枠）。
出典は監査 ID（A〜G）または実装した機能名。

## (a) 今回のスコープで意図的に見送った項目

- **NUL 表記の統一（現状維持で可）** — 低 — 出典: 監査 **D-05**
  取り込み時などの空セル欠損値表現（NUL）の見せ方に統一余地があるが、実害がなく現状維持で問題ないと判断。次回の掃引でまとめて扱う。

- **ミラー窓への「未ヒアリング点線」オプトイン** — 低 — 出典: 機能「未ヒアリング工程ノードの点線描き分け」
  本編キャンバスは未ヒアリング工程を点線枠で描き分ける（`hearingNodeClass` / `.st-unheard`）が、ミラー窓（`MirrorView.tsx`）は未対応。顧客提示用の窓なので、既定 OFF のオプトインとして後日追加する。

- **状況（status）列の CSV / Excel 出力** — 中 — 出典: 機能「ヒアリング状況の編集 UI」＋「Excel 出力の多シート化」
  アウトライン表に状況列を追加したが、Excel/CSV への書き出しには状況列が含まれない。多シート化した工程表シートへ列追加すれば納品物にも進捗が残せる。

- **親工程 status の非対称** — 低 — 出典: 機能「ヒアリング状況の編集 UI」
  `effectiveStatus` / 進捗集計は末端工程のみを母数とし、親工程の status は編集 UI から設定しても集計へ反映されない（意図的な非対称）。親のロールアップ表示が要るか次回に再検討。

- **改善効果集計での milestone 非除外（perRow）** — 低 — 出典: 機能「改善効果レポート」
  `buildCompareReport` の工程別（perRow）行にマイルストーンが混ざりうる。工数集計はマイルストーンを除くが、行の見せ方として除外／注記するか未整理。

- **`round1` 相当の丸めヘルパの集約** — 低 — 出典: 機能「改善効果レポート」
  小数第 1 位丸めが `compareReport.ts` の `round1` を各所（`reportHtml.ts` / `ComparisonDialog.tsx`）で個別に再ラップしている。表示丸めの単一ヘルパへ寄せる掃除枠。

- **quarantine（隔離参照）の修復 UI** — 中 — 出典: 監査（データ整合性枠）
  壊れた参照は読込時に quarantine へ退避して落とさない設計だが、退避内容をユーザーが確認・復旧する UI がない。検証パネルからの導線として自然。

- **インスペクタの課題フォーカス（inspectorIssueFocus）** — 中 — 出典: 機能「検証パネル」
  検証パネルの「方策未記入」等から該当課題へジャンプする際、工程は選択できても課題行までフォーカスを寄せる仕組み（inspectorIssueFocus 相当のシグナル）が未実装。`procedureFocus` と同型で追加できる。

- **検証パネルの常時件数バッジ** — 中 — 出典: 機能「検証パネル」
  検証はダイアログ起動時にのみ走る。ツールバー／ステータスバーに未解決件数の常時バッジを出すと、納品前チェックの発見性が上がる。集計は `lintProject` を購読するだけ。

- **工数 0 を許容するフラグ** — 低 — 出典: 機能「業務リント」
  業務リントは工数未入力（0）を warn として挙げるが、意図的に 0 とする工程（見積対象外など）を除外するプロジェクト単位フラグがない。誤検出が増えるようなら追加。

- **担当未割当→割当の移動可視化** — 中 — 出典: 監査 **A** 系（フロー逆同期）＋カタログ「レーン移動による担当変更の明示トースト」
  未割当レーンから担当レーンへノードを移すと `assigneeId` が無音で書き換わる。担当が付いた変化はトーストで可視化したい（「元に戻す」アクション基盤は今回整備済み）。

- **TableView 結線の DOM テスト（方針上見送り）** — 低 — 出典: 監査（テスト方針）
  表の前工程結線描画は DOM 依存テストが要る箇所だが、「ロジック網羅は core 単体テストへ寄せ E2E は薄く」の方針に沿って今回は見送り。回帰が出たら最小の 1 本のみ追加する。
