# 06. Roadmap（MVP 段階分け）

目玉機能（表編集 → フロー同期）を最短で動かし、段階的に階層・粒度ビュー・出力を足す。

## Phase 1 — 目玉の同期を最小実証
**ゴール**: 単一粒度で「手順一覧表を編集 → 業務フローが自動同期（手動配置は保持）」を動かす。
- 永続化は単一 JSON（アトミック書き込み）。
- コアモデル＋コマンド: 作業の追加／リネーム／削除、担当設定、前後関係（依存）の追加／削除。
- 最小の工程表ビュー（フラットな一覧。階層はまだ単一レベル）。
- フロービュー（`@xyflow/react`）: タスクノードの自動配置、依存からの矢印、**ノードのドラッグ位置が編集を跨いで保持**。
- `reconcileFlow` v1: 自動配置＋孤立検出＋導出エッジ。
- デモ: 表で作業追加／依存を張る → フローにノード・矢印が出る。位置を動かす → リネームしても位置維持。削除 → 確認後に消えて繋ぎ直し。

## Phase 2 — 階層と粒度ビュー
**ゴール**: 大>中>小>詳細の階層と、フローの粒度切替・親範囲バンドを実現。
- コア: `parentId` / `level`、階層操作（子追加・サブツリー移動・粒度フィルタ）。
- 工程表ビュー: 階層アウトライン（WBS 風・展開/折りたたみ）＋粒度フィルタの両対応。
- フロービュー: 粒度セレクタ、**2 軸グリッド（担当レーン × 大/中工程バンド）**、親範囲バンドの導出表示。
- `reconcileFlow` を粒度（scope）対応に拡張。

## Phase 3 — フロー固有構造の保護を強化
**ゴール**: 分岐・合流など図にしか無い構造が同期で壊れない。
- 制御ノード（開始／終了／**判断**／合流）、`pinned` エッジ、分岐条件ラベル。
- `userPathExists` による経路尊重（A→判断→B を直接 A→B で潰さない）。
- レーン移動 → 担当更新（唯一の逆方向同期）。
- undo/redo（コマンド単位のスナップショット復元。`01-architecture.md` §6）。
- 永続化を `.gflow` ZIP バンドルへ拡張＋マイグレーション v1→v2、autosave／競合検知。

## Phase 4 — 入出力（pluggable）
**ゴール**: 既存資産を取り込め、客先提出物を出せる。
- **Excel/CSV 取り込み**（初回ブートストラップ＝新規プロジェクト生成。`Importer` IF・`ImportReport`、`05` §6）。
  既存 Excel 工程表を種にして開始できるようにする。再取り込みマージは対象外。
- **Excel 出力**（現状の納品形態）。様式は実務に合わせて整合（`07` 参照）。
- フロー図の画像出力（PNG/SVG）。
- 入出力はレジストリ（`Importer` / `Exporter` IF）で追加可能に。

## Phase 5 — 仕上げ
- quarantine／修復 UI、大規模案件の性能、テンプレート、（必要なら）フロー PDF 出力。

### 各フェーズ共通の品質ガード
- `reconcileFlow` の**ゴールデンテスト＋冪等性テスト＋プロパティテスト**（`08-testing.md` 参照）を常に緑に保つ。
- `packages/core` は UI 非依存のまま維持し、Node 上で単体テスト可能に保つ。
