# UI/UX 0ベース改善計画（監査統合版・2026-07-04）

7視点並列監査（41所見）を opus が統合。ユーザ目標:「洗練された完璧なプロダクト品質」（過去68案は参考のみ）。

## 総評

骨格（undo/redo・テスト・マイグレーション・ロック・パレット）は高水準。体感が届かないのは
**細部の一貫性と信頼性に穴が集中**しているため: (a)概念の画面間不一致 (b)初回体験の詰め
(c)沈黙する失敗 (d)予測不能なキーボード挙動。「土台は良いのに縁が荒い」。

## テーマ

- T1 二画面同期という約束の一貫性（複数選択・I/O呼称・工数表示が画面ごとに割れる）
- T2 最初の60秒（ツアー経路依存・案内見切れ・空フロー無地）
- T3 沈黙しないシステム（自動退避/バックアップ/ロック失敗が無音・undo無言・保存無反応）
- T4 予測可能な入力と安全な取り消し（フォーカス乗っ取り・Esc不能ドラッグ・負座標行方不明）
- T5 ミクロの仕上げ（0.1666…・「(案A)」残留・AA未達コントラスト・文言ズレ）

## バックログ上位15（優先度順）

1. 空プロジェクトでもスイムレーンの器を常に描く（high/S・`hasLanes` 判定を lanes 定義基準に）
2. 工数表示を共通関数で丸める（high/S・`effortMinutesToHoursDisplay()` を core に1つ、4箇所置換）
3. I/O 呼称を「入力/出力」に統一（high/S・4通りの揺れを一括置換）
4. アウトライン表に複数選択＋一括操作（high/M・FullTable の選択を共有フック化）
5. モーダルの見切れ修正（high/S・max-height:80vh + sticky actions + 長文要約）
6. ノード負座標の行方不明根絶（high/M・move系に下限クランプ＋fitView救出）
7. ドラッグ系を Escape で中断可能に（high/M・共通中断フック）
8. フォーカス乗っ取り（Enter/Delete）停止（high/M・activeElement ガード）
9. 永続化の沈黙する失敗を可視化（high/M・失敗トースト＋StatusBar 直近成功時刻/ロック表示）
10. undo/redo に「何が起きたか」を出す（high/M・履歴ラベル＋トースト＋フラッシュ）
11. オンボーディング全導線化・具体対象ハイライト・案内の可視域中央寄せ（high/M）
12. 行選択モードの主要キー既定ON or 告知（high/M）
13. 複数選択ノードの複製・整列一括対応（mid/M・distribute 純関数）
14. FullTable 狭幅の列重なり修正（high/M・min-width＋横スクロール）
15. 表・フローの roving focus と ARIA（high/L）
