# 開発方針（育成フェーズ・ロードマップ）— 2026-07-03 承認

ユーザ承認済みの全体方針。詳細な経緯・設計判断は各サブプロジェクトの spec を参照。

## 方向

実務投入前の育成フェーズとして「UI/UX 磨き込みの続行＋アプリ内 AI のオプトイン組み込み」を進める。
業務フロー図の根本要件として **マイルストーン**（日付なし・順序ベースの節目）と
**フェーズ帯**（大工程をキャンバス上部の横帯で示す読解ガイド）を追加する。

AI の中核モデル: 現プロジェクト＋ヒアリングメモ → 変更提案リスト → ユーザが 1 件ずつ承認/修正/却下 →
承認分だけ commands 経由で適用（初回=空プロジェクトへの提案、2 回目以降=増分提案）。
「外部送信なし」方針は「**既定は完全オフライン、AI 機能のみ明示オプトイン**」へ更新する（AI サイクルで実施）。

## 確定した設計判断（検証済み）

1. **マイルストーン** = `ProcessTask.kind?: 'milestone'`（マーカータスク）。スキーマ bump 不要。
   ガード: 子を持てない / leafCount・採番から除外 / 専用シェイプ描画。
2. **フェーズ帯** = core の純関数 `derivePhaseStrip(core, view)` で導出（bands.ts と同型・永続化なし）。
   重なり許容（半透明帯）。大工程レベルのビューでは非表示。FlowCanvas と flowSvg の両方で描画。
3. **AI 提案レイヤ** = `BatchOp`/`runBatch` を mcp-server から core へ昇格。AI アダプタは desktop のみ。
   部分承認は desktop 側 ref-DAG リゾルバで下流無効化。適用は 1 スナップショット（undo 一発）。

## ビルド順序

1. **フェーズ帯**（S・独立・モデル変更なし）
2. **マイルストーン**（M・AI より先にガードを固める）
3. **AI 組み込み**（L・安定した op 語彙の上に構築。`runBatch` core 昇格は 2 と並行可）
- 併走: UI/UX カタログ 68 案から各サイクルに数件。

## プロセス

- サブプロジェクトごとに spec → 実装計画 → subagent-driven development の 1 サイクル。
- 実装配役: 機械的=haiku / 標準=sonnet / 複雑な実装=opus。最終ブランチレビュー=opus。
- UI 変更は Playwright MCP での自走確認（実操作＋スクリーンショット）を完了条件に含める。
- フィーチャーブランチ＋PR。マージ判断はユーザ。各サイクル末にユーザが実機評価。

## 対象外・保留

- スケジュール計算（日付・逆算・遅延判定）
- 工程ステータス（ヒアリング進行管理）の拡張・削除 — 現状維持
- Excel 納品出力・画像出力 — 実務投入が近づいた時点で再優先度付け
