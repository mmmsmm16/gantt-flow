# フェーズ帯（大工程ヘッダストリップ）— 設計

日付: 2026-07-03 / ステータス: **取りやめ（2026-07-04）**

> 実装して実機評価した結果、ユーザ判断で取りやめ。既存の親範囲バンド（破線囲い「大: ○○」）で
> フェーズの識別には十分であり、上部常設の帯は情報が重複するため。実装は revert 済み
> （経緯はこの spec と実装計画を履歴として残す）。同種の「上部常設 UI」を再提案する際はこの経緯を参照。

## 目的

フロー図を見たとき「この辺りの工程はどのフェーズ（≒大工程）に属するか」が分かる空間的な読解ガイドを、
キャンバス上部の横長帯として常設する。進捗・状態の強調はしない（工程ステータス機能に依存しない）。

## 仕様

### core（導出・純関数）

- 新規 `packages/core/src/sync/phaseStrip.ts`:
  `derivePhaseStrip(core: Core, view: FlowLevelView): PhaseSegment[]`
  - `PhaseSegment { taskId: Id; label: string; x: number; width: number }`
  - 表示中のタスクノードから祖先をたどり（`bands.ts` と同じ循環ガード付き上昇）、
    **`level === 'large'` の祖先ごと**にメンバーノードの x 範囲（`minX`〜`maxX + SIZE.task.w`、
    左右 PAD 付き）を集計して 1 セグメントにする。
  - `view.level === 'large'` のときは常に `[]`（ノード自体がフェーズのため帯は出さない）。
  - ソートは `x` 昇順 → `taskId.localeCompare`（決定論）。**重なりは許容**（依存順レイアウトでは
    大工程ごとの x 範囲は排他でない。重なりは事実の表現であり、詰め直しはしない）。
  - 永続化しない（`bands.ts` と同じ「ツリーから計算する」導出データ）。

### desktop（画面）

- `FlowCanvas.tsx`: `.lane-rail` と同じ手法の **上部固定レール**（`.flow-scale` の外・
  `position: sticky; top: 0` 相当）。各セグメントを `left: x*scale / width: width*scale` で配置。
  - 半透明の色帯＋ラベル（大工程名）。隣接/重複セグメントが区別できるよう色相 or 濃淡を交互に変える。
  - セグメントが 0 件、または `view.level === 'large'` のときはレール自体を出さない。
  - スコープ付きビュー（`scopeParentId` あり）は単一セグメントに退化してよい（そのフェーズ名の表示になる）。
  - （余力があれば）セグメントクリックで該当大工程のスコープビューへジャンプ。無理には入れない。

### SVG エクスポート

- `flowSvg.ts`: 同じ `derivePhaseStrip` を使い、画像上端に同等の帯（塗り＋ラベル）を描画。
  画面と出力が同じ導出関数を共有することで一致を構造的に保証する。

## 対象外

- 永続化・手動並べ替え・帯のリサイズ
- 「現在フェーズ」の強調・工程ステータス連動
- 大工程間の依存表示の変更（既存 parent bridge のまま）

## テスト方針

`packages/core/test/phaseStrip.test.ts`（golden・`counter()`/`emptyProject()` パターン、
ビューは `reconcileFlow` で構築）:

1. 大2＋各配下に中を持つプロジェクトの medium ビュー → セグメント2件、x 範囲がメンバーを包含、順序決定論。
2. `view.level === 'large'` → `[]`。
3. `scopeParentId` 付きビュー → 単一セグメント。
4. 大工程の x 範囲が交差するレイアウト（手動 x 設定）→ 重なったまま返る（詰めない）。
5. データのみの編集（rename）→ ラベルだけ変わり x/width 不変（位置安定の継承）。
