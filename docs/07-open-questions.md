# 07. Open Questions（後で整合する未確定事項）

設計の骨格は確定済み。以下は実装前後に詰める論点。各所では「TBD」として参照している。

## 1. 手順一覧表の列項目 ✅ 確定（2026-06-06）
- 列セットは確定。詳細は `03-view-spec.md` §1-2、型は `02-data-model.md`（`TaskDetail` / `ProcessTask.code`）。
- 必須（同期）: 工程No・作業名・担当・粒度・前/次工程。
- 標準: 業務内容・インプット・アウトプット・使用システム・工数・備考（I/O はフローにも帳票オブジェクトで表示）。
- 任意: 処理件数/ボリューム・**課題**・**方策**・例外対応・自動化区分・帳票様式/保管・データ連携先・関連規程/統制・作業難易度(H/M/L)。
  - 「課題・改善メモ」は **課題列＋方策列の 2 列に分割**。両列はフローの課題オブジェクトと連動。
- 工数は**時間・0.5h 単位**、工程No は**自動採番＋手動上書き**。
- 残課題: 自動化区分・難易度の選択肢ラベル（手作業/システム自動/一部自動、H/M/L）の最終文言は実装時に微調整。

## 2. Excel 納品様式
- 現状の Excel 納品レイアウト（列順・体裁・粒度ごとのシート構成等）に出力を合わせる。
- `Exporter` IF（`01-architecture.md`）に Excel 実装を追加する Phase 4 で確定。
- 表だけでなくフロー図も Excel に貼るか（画像貼付 or 図形描画）は別途検討。

## 3. フローに出す／出さない項目の最終線引き ✅ おおむね確定（2026-06-06）
- フローに出るオブジェクト: 工程（四角）・**インプット/アウトプット（帳票オブジェクト）**・**課題（赤四角・対象へ注釈線・表示切替）**
  ＋ 制御ノード（開始/終了/判断/合流）。形・色・同期は `03-view-spec.md` §2-3、型は `02-data-model.md` §4。
- 表のみ: 工数・使用システム・どうやって・備考ほか。
- 残課題（軽微）:
  - 入力色/出力色・赤の具体的なカラーパレットと帳票形の見た目（実装時に確定）。
  - 「使用システム」等を工程ノードのツールチップに**補助表示**するかは任意機能として保留。
  - 帳票オブジェクトを「工程ごとに 1 個」ではなく複数（複数帳票）持てるようにするか（当面は I/O 列 1:1）。

## 4. 階層のレベルスキップ可否
- 大の直下に小（中を飛ばす）等を許すか。
- 当面は**許さない前提**で実装（親は常に 1 段細かい粒度）。許容する場合の表示／同期ルールは後で定義。

## 5. 依存（流れ）のスコープと粒度跨ぎ
- 依存は「同一粒度・同一親スコープ内」で張る前提（`02`/`04`）。
- 粒度を跨ぐ流れ（例: ある大工程の最後の小工程 → 次の大工程の最初の小工程）の表現方法は要検討。
  当面はスコープ内に閉じ、跨ぎは上位粒度のフローで表現する。

## 6. 同期の方向性の将来拡張
- 現状の逆方向同期は「レーン移動 → 担当更新」の 1 点のみ。
- フロー上での作業追加（図で直接ステップを足す → コアにタスク生成）を将来サポートするかは要検討
  （サポートする場合は明示的な「作業を追加」操作とする）。
