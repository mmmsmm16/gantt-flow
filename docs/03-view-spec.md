# 03. View Spec

2 つのビュー（工程表ビュー／フロービュー）の仕様。両者は同じコアデータを別の見せ方で表示する。

## 1. 工程表ビュー（手順一覧表）

「どんな業務を・誰が・どれくらいの工数で・どうやって」を行で書く表。**時間軸ガントではない。**

### 1-1. 2 つの見せ方（両対応）
1. **階層アウトライン表（WBS 風）**: 大工程の下に中・小・詳細をインデントで入れ子表示。展開／折りたたみ可能。
2. **粒度フィルタ**: 「大／中／小／詳細」を選んで、その粒度の行だけに絞り込んで一覧。

```
[アウトライン表示]                          [粒度=小 で絞り込み]
▼ 大1 受注処理（営業部）                      受注内容を確認   営業  田中  10分 ...
  ▼ 中1 注文書受付                            在庫を引当       倉庫  佐藤   5分 ...
    ・小1 受注内容を確認  営業 田中 10分        出荷指示         倉庫  佐藤   5分 ...
    ・小2 在庫を引当      倉庫 佐藤  5分
  ▼ 中2 出荷準備
    ・小3 出荷指示        倉庫 佐藤  5分
▼ 大2 ...
```

### 1-2. 列（確定）

**■ 必須（同期対象・コア）** — フローと連動
| 列 | 説明 |
|---|---|
| 工程No | 階層番号（`1` / `1-2` / `1-2-3`）。**自動採番＋手動上書き**（`ProcessTask.code`） |
| 作業名 | 業務・作業の名称（フローのノードラベル） |
| 担当 | 担当者／部署（フローのスイムレーン） |
| 粒度 | 大／中／小／詳細 |
| 前工程／次工程 | 流れ（フローの矢印） |

**■ 標準（表のみ）** — 「どんな業務を・どうやって・どれくらいの工数で」
| 列 | 説明 |
|---|---|
| 業務内容（どうやって） | 手順・方法の説明 |
| インプット | 入力帳票・データ |
| アウトプット | 出力帳票・成果物 |
| 使用システム | システム／ツール |
| 工数 | 所要時間。**時間・0.5h 単位** |
| 備考 | 補足 |

**■ 任意（採用済み・表のみ）**
| 列 | 説明 |
|---|---|
| 処理件数・ボリューム | 1回/月あたりの件数等 |
| 課題・改善メモ | コンサル視点の課題・改善ポイント |
| 例外・イレギュラー対応 | 通常フローから外れる対応 |
| 自動化区分 | 手作業／システム自動／一部自動 |
| 帳票様式・保管 | 様式番号・保管場所/期間 |
| データ連携先 | 次に渡す部署／システム |
| 関連規程・統制 | 規程・マニュアル・内部統制ポイント |
| 作業難易度 | H／M／L |

> すべての「表のみ」列はフローへ**同期しない**（`02-data-model.md` §7）。

### 1-3. 編集操作（コマンド経由）
- 行の追加／削除／リネーム、担当変更、親の変更（インデント変更＝サブツリー移動）、並び替え、前後関係（流れ）設定。
- これらは `commands/*` を通じてコアを変更し、`reconcileFlow` を呼んでフローへ反映（詳細 `04-sync-spec.md`）。

## 2. フロービュー（スイムレーン業務フロー図）

### 2-1. 粒度セレクタ
- 表示する工程粒度（大／中／小／詳細）を切り替える。
- 中・小・詳細では「どのスコープ（どの親工程の中）を見るか」も選べる
  （例: 中工程ビュー = ある大工程の中の中工程フロー）。`FlowLevelView.scopeParentId`。

### 2-2. 2 軸グリッド（独立 2 軸）
**縦＝担当／部署レーン、横＝大>中工程の入れ子バンド。** 両者は独立した軸で、交差してグリッドを作る。

```
                 ┌─ 大工程: 受注処理 ─────────────┬─ 大工程: 出荷処理 ───┐
                 │ ┌中: 注文書受付┐ ┌中: 出荷準備┐ │ ┌中: 梱包出荷────┐ │
 営業 ──────────┤ │(開始)→[受注  │ │           │ │ │              │ │
                 │ │      確認]   │ │           │ │ │              │ │
 倉庫 ──────────┤ │      ↓       │ │[在庫引当] │ │ │[梱包]→[出荷] │ │
                 │ │     [在庫引当]┼→│ →[出荷指示]│ │ │     →(終了)  │ │
                 │ └─────────────┘ └───────────┘ │ └──────────────┘ │
                 └────────────────────────────────┴───────────────────┘
   ↑レーン軸（担当/部署・縦）          ↑バンド軸（大/中工程・横、祖先範囲を背景帯で表示）
```

- **ノード配置**: 各タスクノードは「担当レーン × 祖先バンド」の交点付近に置かれる。
  位置は手動で微調整でき、同期で保持される。
- **親範囲バンドは導出**: 表示中の粒度ノードの祖先をたどり、中工程・大工程ごとに背景帯を描く（`sync/bands.ts`）。
  - 小工程ビュー → 中工程・大工程のバンドを入れ子で表示。
  - 中工程ビュー → 大工程のバンドを表示。
  - 大工程ビュー → バンドなし（最上位）。

### 2-3. ノードの種類
| 種類 | 由来 | 説明 |
|---|---|---|
| タスクノード | コア（同期） | 作業 1 件＝ノード 1 個（その粒度において）。ラベルは作業名。 |
| 開始／終了 (start/end) | フロー固有 | 図にだけ置く端点。 |
| 判断 (decision) | フロー固有 | **分岐はフローでだけ描く**（表には持たせない）。条件は出力エッジのラベルで表現。 |
| 合流 (merge) | フロー固有 | 複数経路の合流。 |

### 2-4. エッジ（矢印）
- コア依存から自動生成される線（`derivedFromDependencyId`）と、ユーザーが描いた線（`pinned`）がある。
- ユーザーが A→判断→B のように経路を作っている場合、同期は直接 A→B を**勝手に張らない**（`04-sync-spec.md`）。

### 2-5. フロー上の編集と逆方向同期
- **ノードを別レーンへドラッグ → 表の「担当」を更新**（唯一の「フロー→表」逆方向同期）。
- それ以外（配置・分岐・合流・コメント）はフロー固有で、コアへは戻さない。

## 3. ビュー間の対応（まとめ）

| コア | 工程表ビュー | フロービュー |
|---|---|---|
| 作業（タスク） | 1 行 | 1 ノード（その粒度） |
| 担当 | 担当列 | スイムレーン |
| 階層（親・粒度） | インデント／粒度フィルタ | 粒度セレクタ＋親範囲バンド |
| 流れ（依存） | 前後関係 | 矢印 |
