# 00. Overview — gantt-flow

## 1. 何を作るか

業務コンサル（業務改善）のヒアリング成果物を、**手順一覧表（工程表）**と
**スイムレーン業務フロー図（工程フロー）**の 2 つの形態で作成し、**両者を同期**させる
デスクトップアプリケーション。

> 目玉機能: **工程表を編集すると、対応する工程フローが自動的に更新される。**
> 同じ業務データを「表」と「図」の 2 つの見せ方で扱い、片方の編集がもう片方へ反映される。

## 2. 誰のための、どんな課題を解くツールか

- **対象ユーザー**: 業務コンサルタント（本人）。客先業務をヒアリングして AS-IS / TO-BE を整理する。
- **現状の課題**: 工程表（Excel 等）と業務フロー図を**別々に**作っているため、
  - どちらかを直すともう一方と食い違う（二重メンテナンス）。
  - 粒度（大／中／小工程）を切り替えて見せるたびに作図し直す手間がかかる。
- **本ツールの狙い**: 単一データから表とフローを生成・同期し、二重メンテナンスを無くす。

## 3. 主要ユースケース

1. ヒアリング結果を**手順一覧表**に入力する（作業・担当・工数・入出力・使用システム等）。
2. 入力に応じて**業務フロー図**が自動生成される。配置・分岐は図の上で整える。
3. 作業の追加・担当変更・順序変更を表で行うと、フローが追従する。
4. **工程の粒度（大／中／小／詳細）を切り替えて**表・フローを閲覧する。
5. 成果物を客先へ提出する（将来: Excel／画像出力。本リポジトリの現段階は設計のみ）。

## 4. 用語定義（重要）

| 用語 | 本ツールでの意味 |
|---|---|
| **工程表** | **手順一覧表型**。「どんな業務を・誰が・どれくらいの工数で・どうやって」を行で書く表。**時間軸のガントチャートではない。** |
| **工程フロー** | **スイムレーン業務フロー図**。担当／部署のレーンに作業ノードを並べ、流れを矢印で、分岐を判断ノードで表す。 |
| **大工程** | **部署・課を跨ぐタイミング**で区切られる、最も粗い粒度の工程。 |
| **中工程** | **帳票・図書ごと**に区切られる工程。 |
| **小工程** | **個人の作業ごと**に区切られる工程。 |
| **詳細工程** | さらに細かく区切られる、最も細かい粒度の工程。 |
| **粒度（レベル）** | 大 > 中 > 小 > 詳細 の 4 段階。**型は固定だが、枝ごとの深さは可変**（小で止まる枝があってよい）。 |
| **スイムレーン（レーン）** | 担当者／部署を表す軸。フロー図の縦方向。 |
| **バンド（帯）** | 大／中工程の範囲を表す軸。フロー図の横方向（祖先範囲の可視化）。 |
| **同期 (reconcile)** | コアデータの変更を各ビューへ反映する処理。 |

## 5. 前提・制約

- **機密データの取り扱い**: 客先機密情報は社内共有フォルダの外へ持ち出せない（社内ルール）。
  → **オフライン前提・ローカルファイル保存・クラウド非依存**のデスクトップアプリとする。
- **本リポジトリの現段階の成果物は「設計書」のみ**（アプリのコードは未実装）。
