## このツールの本質（変えない核）

単一データ（工程＝WBS）から、**工程表**と**スイムレーン業務フロー図**を同期する日本語の社内ツール。
オフライン・ローカルファイル運用。`packages/core` は UI/OS 非依存の純粋ドメイン層で、同期エンジン
（`reconcileFlow`）が心臓部。この設計は美しいので**壊さない**。その上に「製品として完成」と言える
体験・機能・アクセシビリティを足していく。

## 「未完成」を「完成」に変える観点

現状は機能的には驚くほど成熟している（表編集・階層・粒度・同期・I/O・課題・取込・出力・undo・テーマ）。
プロダクトとして詰めるべきは主に **体験の隙間** と **明示要望の積み残し**：

1. **初回体験** — 空の表で始まる。サンプルとオンボーディングで「触ってわかる」状態にする。
2. **発見性・速度** — 検索／ジャンプ／コマンドパレット（Ctrl+K）／ショートカット一覧が無い。
3. **明示要望 #9「スイムレーン可変幅」** — 唯一の未実装要望。並行で太く＋手動リサイズ。要マイグレーション。
4. **フロー編集の完成度** — Delete キー削除・複数選択・自動整列・画面にフィット。
5. **アクセシビリティ** — フォーカスリング・ARIA・キーボード操作・動き低減。
6. **仕上げ** — ステータスバー・堅牢性（エラー境界・自動復旧）・出力品質。

## 作業ログ（新しいものを上に）

### 表のセルカーソル＋Esc 統一＋I/O ショートカット＋ミニマップ切替（`feature/keyboard-first` 続き）
- **表のセル単位ナビゲーション**: h/l・←→ で列カーソルを移動し、Enter で「そのセル」を編集
  (作業名だけでなく担当・工数・粒度・備考なども)。アウトライン/全項目表共通
  (useRowSelectionKeys に列カーソル、セルは行内の data-cell 属性で特定)。
- **Esc の統一**: 編集中の Esc は「編集をやめて選択モードへ」(blur・選択維持)。表・フロー共通。
  もう一度 Esc で選択解除。
- **Enter での行挿入を廃止**(誤挿入防止): 名前入力の Enter は確定して選択モードへ。
  行追加は n / Ctrl+Enter / ＋ボタン。確定時の伝播でセル編集が再発火するバグも修正。
- **ヒアリングステータスの UI を撤去**(インスペクタ/全項目表の列・絞り込み/ステータスバー/
  パレット)。データモデルは旧ファイル互換のため温存。
- **フローの I/O ショートカット**: 選択工程に i=インプット / o=アウトプット追加
  (Alt+I/O は常時有効の代替)。
- **ミニマップの表示/非表示**: 設定(一般)とパレットで切替・永続化(既定 ON)。

### スコープ既定=全体＋親接続の同期ずれ解消＋接続モードの方向選択（`feature/keyboard-first` 続き）
- **フローの既定スコープを「全体」に**: 粒度切替・サンプル・テンプレートの初期表示は常に
  全体俯瞰。同一粒度の setLevel はスコープを保ち、表クリック/ジャンプは「全体で俯瞰中は
  維持・特定スコープ中のみ文脈(親)へ追従」。
- **同期ずれ解消**: 大工程同士が繋がっているとフローは子の「末端→先頭」をブリッジ表示
  するが工程表には出ていなかった。導出を deriveParentBridges として core に一元化し
  (reconcileFlow も同じ関数を使用=構造的に一致)、表の前工程列・全項目表・インスペクタに
  「⤷ 工程名」(破線・読み取り専用)として表示。core テスト 4 本。
- **接続モード(c)の方向選択**: Tab 巡回に加えて 矢印 / hjkl で「押した方向の最近傍候補」を
  選べるように(空間ナビと同じ感覚)。ヒント文言も更新。

### フローのキーボード選択(空間ナビ)＋詳細パネルの分離（`feature/keyboard-first` 続き）
ユーザー指摘「未選択だと矢印で移動できる状態にならない / クリックすると詳細パネルが出て邪魔」。
- **矢印=選択を隣のノードへ移す(空間ナビ)、Alt+矢印=ノードを移動(8px・Shift+Alt=32px)** に
  変更し、表(↑↓=行選択 / Alt+↑↓=行移動)と操作体系を統一。未選択で矢印を押すと左上の
  ノードから開始。spatialNav.ts(方向の最近傍・純関数)を新設、選択先へスクロール追従。
- **選択と詳細パネルを分離**(useUI.inspectorOpen): フローのクリック/キーボード選択は
  ハイライトのみ。選択済みノードの再クリック(または再 Enter)で詳細パネルを開く。
  表の行クリック・パレットのジャンプは従来どおり即表示。× で閉じても選択は維持。
- テスト: spatialNav 5 本 + keymap 更新(計 desktop 64)。実機確認: 未選択→矢印で選択開始、
  Alt+矢印 8/32px 移動、再クリックで詳細、×後も選択維持、表クリックは従来動作。

### 矢印のノード回避＋色のクイックショートカット（`feature/keyboard-first` 続き）
- **矢印の経路がノードを避ける**: 線が工程の上を通ると「繋がっている」ように誤読される
  問題への対応。core/sync/edgeRoute を新設し、縦の通り道(midX)候補を障害物の脇から
  生成して交差数で採点、避け切れないときは横の通り道で迂回(HVHVH)。画面/SVG/PNG/印刷の
  全レンダラ共有。ラベル・矢印ツールバーも経路に追従。純関数テスト 7 本。
  実機確認: 受注確定→請求書作成の矢印が出荷指示を貫通せず左脇で下りて空き行を通る。
- **色のクイックショートカット**: 塗り色 Alt+1(なし)/2(青)/3(赤)、文字色 Alt+Shift+1/2/3。
  Mac の Option+数字対策で e.code(Digit*)判定。Alt 付きのためシングルキー OFF でも有効。
  ヘルプ/設定に「工程カラー」グループとして掲載(キー変更可)。

### キーボード・ファースト（`feature/keyboard-first`）— マウスレス操作とカスタマイズ
コンサルタントの高速入力のため、マウスを使わずに大半の操作を完結できるようにする一連の機能。
- **キーマップ基盤**: `keymap.ts` を単一の真実とし、照合・g リーダー(1秒)・ユーザー上書きを純粋関数で提供。
  ディスパッチは `useGlobalHotkeys` に一元化(window keydown は原則 1 本。IME / オーバーレイ / 編集中ガード)。
  `useUI.activePane`(表/フロー)で単キーをルーティングし、Delete/Esc の二重発火を構造的に防止。
- **表の行選択モード**: j/k・↑↓ で行選択移動、Enter/F2=編集、n=次に追加、Shift+N=子追加、
  Alt+↑↓=並べ替え、Tab=字下げ/字上げ(行選択中のみ)、Ctrl+D=複製、Delete=削除、Space=折りたたみ。
  アウトラインと全項目表で共有(`useRowSelectionKeys`)。
- **グローバル**: u=undo / v=表モード切替 / `/`=パレット / Ctrl+1/2・F6=ペイン切替 /
  g t・g f・g i・g s・g 1〜4(画面移動・粒度)。Ctrl+P=印刷も追加。
- **パレット引数コマンド(12 個)**: コマンド選択 → 入力欄が引数モードに(チップ表示・Esc で戻る)。
  担当/工数/粒度/工程名/課題/方策/I/O/前工程/スコープ/付箋/接続先。候補はファジー検索＋自由入力。
- **フローのキーボード操作**: 矢印/hjkl=ノード移動(8px、Shift=32px)、+/-/0/f=ズーム、Enter/F2=リネーム、
  **c=接続モード**(候補を距離順に Tab/矢印で循環 → Enter で接続。マウス接続と同じハイライトを共用)。
- **ショートカットの個人カスタマイズ**: 設定ダイアログでキーの変更(打鍵キャプチャ)・無効化・既定リセット。
  localStorage 永続。実効キーマップをヘルプ(?)・ステータスバーのヒントと共有=表示と動作が常に一致。
- ステータスバーにアクティブペイン別ヒントと g リーダー待機チップ。keymap/suggestions のテスト 20 本追加。
- 実機スモーク確認済み: j/k 行移動・引数コマンド(担当設定)・g リーダー・ノード移動(8/32px)・接続モード。

### 使いやすさ強化バッチ（調査 → 改善提案 21 項目を順に実装）

  非対応ブラウザはダウンロードにフォールバック）。「名前を付けて保存」。最近開いたファイル
 （IndexedDB にハンドル永続）。保存ごとの世代バックアップ（5 世代・復元ダイアログ）。
- **編集の速度**: 複数選択と一括操作（フロー: Shift+ドラッグ範囲選択→一括移動/削除、
  表: Ctrl/Shift+クリック→担当一括設定/一括削除）。行の複製（Ctrl+D）。クリップボード
  貼り付けで複数行追加。表の絞り込み（担当/課題/工数未入力/自動化/ステータス）。
- **フロー操作**: 矢印選択でミニツールバー（ラベル/削除）。制御ノード・付箋をパレットから
  画面中央に配置。空白ダブルクリックで工程作成、ノードのダブルクリック/F2 でその場リネーム。

- **成果物**: 印刷 / PDF（工程表＋フロー図、A4 横）。PNG 出力。SVG/PNG にタイトル・出力日・
  凡例。課題一覧ビュー（横断一覧＋Excel 出力）。サマリ（担当別工数・自動化割合・粒度別）。
- **立ち上がり**: 業務テンプレート 4 種（受注〜出荷/月次決算/購買/入社手続き）。初回サンプル時の
  使い方ツアー（4 ステップ・操作を妨げないコーチマーク）。
- **その他**: 帳票名オートコンプリート（表記ゆれ防止）。工程ステータス（ヒアリング進行管理）。
  全項目表の逐次レンダリング（大規模対応）。取り込み中スピナー。出所照合の正規化。
- テスト: core 88 / desktop 28 すべて緑・型・ビルド緑を維持。

### バグ修正（本命） — レーン名が下方向に累積ずれ（CSS 退行）
ユーザー指摘「経理部に工程が入っていない（太いレーンで担当名が帯からずれる）」の**真因**。
- 原因: レーン手動リサイズ機能を足したとき `.lane-label { position: relative; }` を追加してしまい、
  既存の `position: absolute`（＝レーン名の絶対配置）を**上書き**。結果、レーン名が流し込み配置になり
  **先行レーンの高さぶん下へ累積ずれ**（太いレーンほど顕著で、担当名が空き帯に落ちて見えた）。
- 見落とし理由: 座標ダンプで `style.top`（CSS 値＝正しい）を見ていて、`getBoundingClientRect`（実描画位置＝ずれ）
  を見ていなかった。実画面の矩形を取って初めて露見（与信確認@466 に対し経理部ラベル@698）。

### #9 スイムレーン可変高さ（明示要望・実装完了）
要望: 工程が並行するときレーンを並行ぶん太く（逐次なら太くしない）＋手動ドラッグでも広げられる。

**設計判断（積み残しメモの論点に回答）**:
- **保存形式**: `Swimlane.height?` を**任意追加**。旧ファイルは未指定＝既定で読めるので**マイグレーション不要**
  （破壊的変更でないため版は上げない）。`schema.ts` も optional で追従。
- **タスク y は絶対座標のまま**（レーン相対にしない＝モデル変更を避け reconcile 不変条件を保つ）。
  レーンを伸縮したら**下のレーンのノードを同量シフト**して整合（`store.setLaneHeight`）。
- **並行→太く**は**整列（tidy）時に判定**: 同レーン×同段に同居する工程を「並行」とみなしサブ行へ縦積みし、
  レーン高さを並行度に合わせて拡縮（逐次＝別段なので太くならない）。**reconcile では太くしない**
  （「データ編集で x/y 不変」の不変条件を守るため。自動整列という明示操作に寄せる）。
- レーン幾何（累積上端・基準 y・最近傍レーン判定）を `core/sync/lanes.ts` に**一元化**し、
  reconcile・tidy・画面・SVG出力の 4 箇所が共有。
- UI: ラベル列のグリップをドラッグで手動リサイズ（プレビュー線→確定で 1 undo）。
- テスト: 並行で太く/逐次で据え置き、手動リサイズの連動シフト＋undo。全レンダラ整合。

### フロー編集の充実 — 整列 / 全体表示 / 選択＋Delete
- **自動整列（整列ボタン）**: 依存の前後で左→右に段組み（longest-path）＋担当レーンで縦配置。
  `core/sync/tidy.ts` に純粋関数を追加（reconcile と違い「手動配置を保持しない」明示操作）。テスト4本。
- **全体表示（全体ボタン）**: 全ノードの外接矩形に合わせて倍率＋スクロール（拡大は100%まで）。
- **選択モデル＋Delete**: 制御ノード/付箋/矢印をクリックで選択（アクセント枠）、Delete で削除、Esc で解除。
- **矢印削除の意味を是正**: 導出エッジ（依存由来）を消すと、元の前後関係（依存）も消える＝表に反映し
  再同期でも復活しない。手動エッジは図からのみ除去。矢印はダブルクリックで分岐ラベル編集に変更。
