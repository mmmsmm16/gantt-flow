# gantt-flow を「完成したプロダクト」にする — Claude の構想と作業ログ

> このブランチ（`feature/release-ready`）は、「自分（Claude）だったらここまで仕上げてリリースする」という
> レベルを目指した再構築・改修・機能拡張の記録です。**まるごと採用しても、部分的に参考にしても構わない**
> 形で進めます。各変更はテスト緑・型チェック緑・ビルド緑を維持し、1 機能 = 1 コミットで積みます。

## このツールの本質（変えない核）

単一データ（工程＝WBS）から、**工程表**と**スイムレーン業務フロー図**を同期する日本語の社内ツール。
オフライン・ローカルファイル運用。`packages/core` は UI/OS 非依存の純粋ドメイン層で、同期エンジン
（`reconcileFlow`）が心臓部。この設計は美しいので**壊さない**。その上に「製品として完成」と言える
体験・機能・アクセシビリティを足していく。

## 「未完成」を「完成」に変える観点

現状は機能的には驚くほど成熟している（表編集・階層・粒度・同期・I/O・課題・取込・出力・undo・テーマ）。
プロダクトとして詰めるべきは主に **体験の隙間** と **明示要望の積み残し**：

1. **初回体験** — 空の表で始まる。サンプルとオンボーディングで「触ってわかる」状態にする。
2. **発見性・速度** — 検索／ジャンプ／コマンドパレット（Ctrl+K）／ショートカット一覧が無い。
3. **明示要望 #9「スイムレーン可変幅」** — 唯一の未実装要望。並行で太く＋手動リサイズ。要マイグレーション。
4. **フロー編集の完成度** — Delete キー削除・複数選択・自動整列・画面にフィット。
5. **アクセシビリティ** — フォーカスリング・ARIA・キーボード操作・動き低減。
6. **仕上げ** — ステータスバー・堅牢性（エラー境界・自動復旧）・出力品質。

## 作業ログ（新しいものを上に）

### 全項目表: アクティブセル/行の明示＋行操作コマンド（ユーザー希望）
- **今いる場所を明示**: フォーカス中のセルをアクセントの太枠で（Excel 風）、その行を地色＋No. にアクセントバーで
  強調（`:focus-within` ベース。クリック選択行も同じ強調）。
- **行の追加/削除をコマンドで**: コマンドパレット（Ctrl/⌘+K）に「工程を追加（選択の次に）」「選択中の工程を削除」。
  全項目表では **Ctrl/⌘+Enter＝行追加 / Ctrl/⌘+Delete＝行削除（確認あり）**。ヘルプ一覧・ヒント文も更新。

### 全項目表の使い勝手強化（ユーザー希望）
- **前工程の「＋」を縮小**（インプット列の＋と同サイズ。`appearance:none` の小さなセレクト）→ 行が太らない。
- **テキスト列を折り返し全文表示**: 業務内容/使用システム/課題/方策/備考ほかを自動伸縮 textarea に
  （`field-sizing` 非依存の JS 自動グロー、最大 220px で内部スクロール）。
- **列幅をドラッグ調整**: ヘッダ右端のハンドルで幅変更（`colgroup` で制御、`useUI.ftColWidths` に永続）。
  先頭固定列の left は実幅から動的計算。
- **行操作の UX**: アクション列を右端固定（行追加＋/削除🗑）、行クリックで選択（先頭にアクセントバー）、
  作業名で Enter＝次の行を追加、データ列で Enter＝同列の下のセルへ移動。ヒント文を追加。

### 全項目表に列の表示切替＋並べ替え（ユーザー希望）
- **列の表示/非表示**: 列メニューで任意の列をトグル（粒度列・全データ列）。localStorage 永続。
  先頭固定列（No.＋表示中の粒度列）の left はインラインで動的計算し、列を隠しても固定が崩れない。
- **並べ替え**: ヘッダクリックで工数・担当・難易度・自動化・各粒度名で昇順/降順/解除をサイクル。
  並べ替え中も各行は自分の祖先（大/中/小）を表示。「並べ替え解除」で工程順に戻る。

### 新機能 — 全項目フル表（完全な工程表ビュー）
ユーザー要望「工程表を完全に表として表示したい」への対応。アウトラインビューとタブ切替の新ビュー。
- 全工程を行・全フィールドを列にした 1 枚の編集グリッド（`FullTable.tsx`）。その場で全項目を編集。
- **粒度は 大工程/中工程/小工程/詳細工程 の 4 列**で表現（粒度セレクタ廃止）。各行は自分の粒度の列に
  作業名を入力し、上位粒度の列には祖先（親）の工程名を再掲（同じ大工程の中工程は同じ大工程名）。
- **課題と方策は独立した列**。I/O はインプット/アウトプット列、前工程はピル＋追加。
- ヘッダ固定＋先頭（No.＋4粒度列）固定で、横スクロールしても階層の文脈が見える。
- 表示モードは `useUI.tableMode`（outline / full）。full のときはフロー/インスペクタを畳んで全幅。

### 調整 — 工程ノードの初期配置をレーン帯の中央に（ユーザー希望）
「初期配置が少し高い、帯の中央にしたい」への対応。`LANE_BASE_Y` を固定 40 から
`帯上端 + (帯高さ - 工程ノード高さ)/2`（既定 120 のとき 62）に変更し、工程が帯の中央に来るように。
reconcile/tidy/最近傍レーン判定/手動リサイズはすべて `laneTaskBaseY` 経由なので整合維持。テスト座標も更新。

### 調整 — 担当ラベルはレーン中央寄せ（ユーザー希望）
ドリフト不具合の修正後、担当名はレーンの帯の**中央**に置く（上寄せから戻す）。画面/SVG 両方。

### バグ修正（本命） — レーン名が下方向に累積ずれ（CSS 退行）
ユーザー指摘「経理部に工程が入っていない（太いレーンで担当名が帯からずれる）」の**真因**。
- 原因: レーン手動リサイズ機能を足したとき `.lane-label { position: relative; }` を追加してしまい、
  既存の `position: absolute`（＝レーン名の絶対配置）を**上書き**。結果、レーン名が流し込み配置になり
  **先行レーンの高さぶん下へ累積ずれ**（太いレーンほど顕著で、担当名が空き帯に落ちて見えた）。
- 見落とし理由: 座標ダンプで `style.top`（CSS 値＝正しい）を見ていて、`getBoundingClientRect`（実描画位置＝ずれ）
  を見ていなかった。実画面の矩形を取って初めて露見（与信確認@466 に対し経理部ラベル@698）。
- 修正: その `position: relative` 上書きを削除（元の absolute が手動リサイズ・ハンドルの基準にもなる）。
  → 全レーンで名前が帯に正しく一致。テスト・ビルド緑、実機の実描画矩形で一致を確認。

### フォロー — 太いレーンの見やすさ（名前位置・帯・空名ノード）
「整列で太くなったレーンで、担当名が無い帯/工程がレーンからずれて見える」への対応。実機の座標を
ダンプして確認したところ**配置自体は正しい**（各工程は担当レーンの帯内に収まっている）が、見え方が紛らわしかった:
- **レーン名を上寄せ**に（中央寄せだと太いレーンで先頭/末尾の行に名前が無いように見えた）。画面/SVG両方。
- **帯の縞を少し濃く**して、太いレーンでも 1 つの帯だと分かるように（薄すぎた）。
- **フロー作成の工程に既定名「新規工程」**を付与（空名の白箱を作らない＝名前の無い帯に見える誤解を防ぐ）。

### バグ修正 — 担当者名の無いスイムレーンを出さない
ユーザー指摘「担当者名が書かれていないレーンがある（幅調整の影響か）」への対応。
- 原因: 工程に担当が無いビュー（特に **大/全体**＝大工程は担当を持たない）で、フォールバックの
  単一レーン「（未割当）」を描いていた＝担当者名の無いレーンに見えていた。
- 修正: **担当レーンが 1 本も無いビューではスイムレーン枠自体を描かない**（ラベル列・帯・区切り線・
  「（未割当）」を出さず、工程ノードだけを素のキャンバスに表示）。画面/SVG出力の両レンダラに適用。
- 全ビューの担当レーン（営業部/経理部/…）の表示・可変高さ・手動リサイズは従来どおり正常（実機で確認）。
  flowSvg のレーン描画にテスト追加。

### フィードバック対応 — 工程表の親子関係を見やすく
ユーザー指摘「工程表で親子の関係が認識しづらい」への対応。
- **ツリーガイド（VS Code 風）**: 各行に縦の接続線＋エルボー（├/└）を重ね、どの工程がどの親の子かを
  一目で追えるように（`buildOutline` で祖先の継続線/末子判定を計算、名前セルに絶対配置で全高に描画）。
- **親行の強調**: 親工程は作業名を太字＋淡いアクセント地色で「まとまりの先頭」を明示。
- インデントを 18→22px に拡大、選択行はガイドをアクセント色に。

### 堅牢性・仕上げ — 自動復旧 / エラー境界 / 閉じる前確認 / 表の可読性
- **自動復旧（autosave）**: 未保存の変更を localStorage にデバウンス退避し、起動時に「前回の作業を
  復元しますか？」を提案。ファイル保存/新規/開く で退避データを消す。復元は未保存(dirty)扱いで保存を促す。
- **エラー境界**: 描画時の例外で白画面にせず、復旧画面（作業をファイルに退避 → 再読み込み）を出す。
- **閉じる前の確認**: 未保存のままタブ/ウィンドウを閉じようとしたら警告（beforeunload）。
- **工程表の可読性**: `table-layout: fixed`＋表示列に応じた `min-width` で、狭いときは作業名を潰さず
  ペインを横スクロール（従来は名前が「受」等に潰れていた）。
- 実機検証: autosave 書き込み（12768B/16工程）、復元プロンプト表示、作業名フル表示を確認。

### #9 スイムレーン可変高さ（明示要望・実装完了）
要望: 工程が並行するときレーンを並行ぶん太く（逐次なら太くしない）＋手動ドラッグでも広げられる。

**設計判断（積み残しメモの論点に回答）**:
- **保存形式**: `Swimlane.height?` を**任意追加**。旧ファイルは未指定＝既定で読めるので**マイグレーション不要**
  （破壊的変更でないため版は上げない）。`schema.ts` も optional で追従。
- **タスク y は絶対座標のまま**（レーン相対にしない＝モデル変更を避け reconcile 不変条件を保つ）。
  レーンを伸縮したら**下のレーンのノードを同量シフト**して整合（`store.setLaneHeight`）。
- **並行→太く**は**整列（tidy）時に判定**: 同レーン×同段に同居する工程を「並行」とみなしサブ行へ縦積みし、
  レーン高さを並行度に合わせて拡縮（逐次＝別段なので太くならない）。**reconcile では太くしない**
  （「データ編集で x/y 不変」の不変条件を守るため。自動整列という明示操作に寄せる）。
- レーン幾何（累積上端・基準 y・最近傍レーン判定）を `core/sync/lanes.ts` に**一元化**し、
  reconcile・tidy・画面・SVG出力の 4 箇所が共有。
- UI: ラベル列のグリップをドラッグで手動リサイズ（プレビュー線→確定で 1 undo）。
- テスト: 並行で太く/逐次で据え置き、手動リサイズの連動シフト＋undo。全レンダラ整合。

### フロー編集の充実 — 整列 / 全体表示 / 選択＋Delete
- **自動整列（整列ボタン）**: 依存の前後で左→右に段組み（longest-path）＋担当レーンで縦配置。
  `core/sync/tidy.ts` に純粋関数を追加（reconcile と違い「手動配置を保持しない」明示操作）。テスト4本。
- **全体表示（全体ボタン）**: 全ノードの外接矩形に合わせて倍率＋スクロール（拡大は100%まで）。
- **選択モデル＋Delete**: 制御ノード/付箋/矢印をクリックで選択（アクセント枠）、Delete で削除、Esc で解除。
- **矢印削除の意味を是正**: 導出エッジ（依存由来）を消すと、元の前後関係（依存）も消える＝表に反映し
  再同期でも復活しない。手動エッジは図からのみ除去。矢印はダブルクリックで分岐ラベル編集に変更。

### 発見性・操作性 — コマンドパレット / ステータスバー / ショートカット
- **コマンドパレット（Ctrl/⌘+K）**: アクション実行＋工程名/工程No でのファジー検索ジャンプ。
  これまで検索手段が皆無だったので、大きな案件でも目的の工程に一発で飛べる。
- **ステータスバー**: 工程数（粒度別内訳）・担当数・合計工数・表示中ビュー・保存状態を常時表示。
- **ショートカット一覧（?）**: 既存の隠れた操作を一覧化。発見性とアクセシビリティ。
- **アクセシビリティ**: フォーカスリングを全要素へ拡張、`prefers-reduced-motion` で動きを抑制、
  共有 `kbd` 表記。Ctrl/⌘+S が保存済み状態（dirty 解除）を反映するよう修正。
