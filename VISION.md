## このツールの本質（変えない核）

単一データ（工程＝WBS）から、**工程表**と**スイムレーン業務フロー図**を同期する日本語の社内ツール。
オフライン・ローカルファイル運用。`packages/core` は UI/OS 非依存の純粋ドメイン層で、同期エンジン
（`reconcileFlow`）が心臓部。この設計は美しいので**壊さない**。その上に「製品として完成」と言える
体験・機能・アクセシビリティを足していく。

## 「未完成」を「完成」に変える観点

現状は機能的には驚くほど成熟している（表編集・階層・粒度・同期・I/O・課題・取込・出力・undo・テーマ）。
プロダクトとして詰めるべきは主に **体験の隙間** と **明示要望の積み残し**：

1. **初回体験** — 空の表で始まる。サンプルとオンボーディングで「触ってわかる」状態にする。
2. **発見性・速度** — 検索／ジャンプ／コマンドパレット（Ctrl+K）／ショートカット一覧が無い。
3. **明示要望 #9「スイムレーン可変幅」** — 唯一の未実装要望。並行で太く＋手動リサイズ。要マイグレーション。
4. **フロー編集の完成度** — Delete キー削除・複数選択・自動整列・画面にフィット。
5. **アクセシビリティ** — フォーカスリング・ARIA・キーボード操作・動き低減。
6. **仕上げ** — ステータスバー・堅牢性（エラー境界・自動復旧）・出力品質。

## 作業ログ（新しいものを上に）

### フィードバック対応 — 工程表の親子関係を見やすく
ユーザー指摘「工程表で親子の関係が認識しづらい」への対応。
- **ツリーガイド（VS Code 風）**: 各行に縦の接続線＋エルボー（├/└）を重ね、どの工程がどの親の子かを
  一目で追えるように（`buildOutline` で祖先の継続線/末子判定を計算、名前セルに絶対配置で全高に描画）。
- **親行の強調**: 親工程は作業名を太字＋淡いアクセント地色で「まとまりの先頭」を明示。
- インデントを 18→22px に拡大、選択行はガイドをアクセント色に。

### アクセシビリティ深掘り
- **スキップリンク**（工程表へ）＋ランドマーク（banner / 名前付き region）。
- **フロー図のキーボード操作**: 工程・制御・付箋を Tab で巡回（role=button / aria-label / aria-pressed）、
  Enter/Space で選択、Delete で削除（マウス前提だった図を一部キーボードでも操作可能に）。
- **レーン高さのキーボード操作**: グリップを role=separator にし、上下キーで高さ変更（Shift で大きく）。
- **オーバーレイのフォーカストラップ**: コマンドパレット / ヘルプ / モーダルで Tab を内部に閉じ込め、
  確認ダイアログは確定ボタンへ自動フォーカス。
- フォーカスリングを keyboard 時のみ表示（:focus-visible）。実機で Tab フォーカス可視化を確認。

### 総仕上げ — 自己レビュー / 評価ガイド
- レビュー修正: レーンリサイズのプレビューをガイド線のみに（ドラッグ中のレーン/ノードのズレを解消）、
  復元提案を 1 セッション 1 回に（StrictMode 二重実行対策）、パレットのキーワード誤記修正、
  「フローを整列」をコマンドパレットへ追加。
- `RELEASE-NOTES.md` を追加（採用評価用の触り方・変更一覧・品質・設計約束）。
- 全テスト緑（core 75 / desktop 21）・型・ビルド緑を最終確認。実機スモークで回帰なしを確認。

### 堅牢性・仕上げ — 自動復旧 / エラー境界 / 閉じる前確認 / 表の可読性
- **自動復旧（autosave）**: 未保存の変更を localStorage にデバウンス退避し、起動時に「前回の作業を
  復元しますか？」を提案。ファイル保存/新規/開く で退避データを消す。復元は未保存(dirty)扱いで保存を促す。
- **エラー境界**: 描画時の例外で白画面にせず、復旧画面（作業をファイルに退避 → 再読み込み）を出す。
- **閉じる前の確認**: 未保存のままタブ/ウィンドウを閉じようとしたら警告（beforeunload）。
- **工程表の可読性**: `table-layout: fixed`＋表示列に応じた `min-width` で、狭いときは作業名を潰さず
  ペインを横スクロール（従来は名前が「受」等に潰れていた）。
- 実機検証: autosave 書き込み（12768B/16工程）、復元プロンプト表示、作業名フル表示を確認。

### #9 スイムレーン可変高さ（明示要望・実装完了）
要望: 工程が並行するときレーンを並行ぶん太く（逐次なら太くしない）＋手動ドラッグでも広げられる。

**設計判断（積み残しメモの論点に回答）**:
- **保存形式**: `Swimlane.height?` を**任意追加**。旧ファイルは未指定＝既定で読めるので**マイグレーション不要**
  （破壊的変更でないため版は上げない）。`schema.ts` も optional で追従。
- **タスク y は絶対座標のまま**（レーン相対にしない＝モデル変更を避け reconcile 不変条件を保つ）。
  レーンを伸縮したら**下のレーンのノードを同量シフト**して整合（`store.setLaneHeight`）。
- **並行→太く**は**整列（tidy）時に判定**: 同レーン×同段に同居する工程を「並行」とみなしサブ行へ縦積みし、
  レーン高さを並行度に合わせて拡縮（逐次＝別段なので太くならない）。**reconcile では太くしない**
  （「データ編集で x/y 不変」の不変条件を守るため。自動整列という明示操作に寄せる）。
- レーン幾何（累積上端・基準 y・最近傍レーン判定）を `core/sync/lanes.ts` に**一元化**し、
  reconcile・tidy・画面・SVG出力の 4 箇所が共有。
- UI: ラベル列のグリップをドラッグで手動リサイズ（プレビュー線→確定で 1 undo）。
- テスト: 並行で太く/逐次で据え置き、手動リサイズの連動シフト＋undo。全レンダラ整合。

### フロー編集の充実 — 整列 / 全体表示 / 選択＋Delete
- **自動整列（整列ボタン）**: 依存の前後で左→右に段組み（longest-path）＋担当レーンで縦配置。
  `core/sync/tidy.ts` に純粋関数を追加（reconcile と違い「手動配置を保持しない」明示操作）。テスト4本。
- **全体表示（全体ボタン）**: 全ノードの外接矩形に合わせて倍率＋スクロール（拡大は100%まで）。
- **選択モデル＋Delete**: 制御ノード/付箋/矢印をクリックで選択（アクセント枠）、Delete で削除、Esc で解除。
- **矢印削除の意味を是正**: 導出エッジ（依存由来）を消すと、元の前後関係（依存）も消える＝表に反映し
  再同期でも復活しない。手動エッジは図からのみ除去。矢印はダブルクリックで分岐ラベル編集に変更。

### 発見性・操作性 — コマンドパレット / ステータスバー / ショートカット
- **コマンドパレット（Ctrl/⌘+K）**: アクション実行＋工程名/工程No でのファジー検索ジャンプ。
  これまで検索手段が皆無だったので、大きな案件でも目的の工程に一発で飛べる。
- **ステータスバー**: 工程数（粒度別内訳）・担当数・合計工数・表示中ビュー・保存状態を常時表示。
- **ショートカット一覧（?）**: 既存の隠れた操作を一覧化。発見性とアクセシビリティ。
- **アクセシビリティ**: フォーカスリングを全要素へ拡張、`prefers-reduced-motion` で動きを抑制、
  共有 `kbd` 表記。Ctrl/⌘+S が保存済み状態（dirty 解除）を反映するよう修正。
